# local_mycoursesfilter

`local_mycoursesfilter` is a Moodle local plugin that renders a dedicated, linkable
“My courses” page for the current user and applies filter, category, sort,
display, title, and return navigation options from URL parameters.

It is intended for dashboard shortcuts, course navigation links, deep links from
other local plugins, and portal-style integrations where a pre-filtered course
overview is useful.

## Current development status

The plugin is already usable in production-oriented Moodle instances.
Its current state is best described as **integration-ready beta**:
feature-complete for the intended use case, but still evolving in ergonomics and
coverage for edge cases.

### What is already in good shape

- dedicated standalone page at `/local/mycoursesfilter/index.php`
- layout closely aligned with `my/courses.php`
- Mustache-based rendering for card, list, and summary views
- safe handling for `returnurl`, including `returnurl=this`
- category shortcuts with recursive expansion
- configurable default category recursion behaviour
- configurable default title and optional title override via URL
- optional toolbar persistence mode using **core** `my/courses` preferences
- no plugin-owned database tables
- PHPUnit and Behat coverage for core request and navigation helpers
- CI-ready structure for Moodle plugin validation

### Current limitations

- toolbar persistence can only map **compatible** states to core `my/courses`
  preferences; plugin-specific filters like `notstarted` and `completed` remain
  request-only even when persistence is enabled
- course-state buckets depend on available Moodle completion and access metadata
- the plugin is still an integration plugin, not a full replacement for every
  core `my/courses` behaviour

## Security and production suitability

The plugin is suitable for productive Moodle systems when links are generated by
trusted Moodle code or trusted site integrations.

### Positive security properties

- request parameters are normalised with Moodle `optional_param()` cleaning or
  strict token parsing
- category selectors accept only numeric ids and the keywords `this`, `parent`,
  and `children`
- category recursion uses Moodle DML with bound parameters only
- `coursename` and `customfield` matching use case-insensitive partial matching
  without raw SQL interpolation
- explicit `returnurl` values are restricted to local URLs
- `returnurl=this` only resolves when the referrer belongs to the same Moodle
  site
- the page remains protected by the capability `local/mycoursesfilter:view`

### Operational recommendations

- generate links from Moodle renderers, blocks, local plugins, or trusted portal
  code instead of letting untrusted users compose arbitrary URLs manually
- keep PHPUnit and Behat checks mandatory in CI for future changes
- review the practical meaning of `notstarted`, `inprogress`, and `completed`
  against the completion model used on your site

## Privacy

The plugin does **not** store its own personal data and does **not** create its
own database tables.

If toolbar persistence is disabled, the page is fully request-driven.
If toolbar persistence is enabled, the plugin reuses Moodle core
`block_myoverview` user preferences for compatible sort, grouping, and display
states instead of introducing plugin-owned preferences.

A Privacy API provider is included and declares that the plugin stores no own
personal data.

## Feature overview

- shows only courses in which the current user is enrolled
- supports partial course-name matching via `coursename`
- supports category filtering, including comma-separated ids and contextual
  keywords
- supports tag filtering by tag id or name
- supports combined custom field filtering via `customfield`
- supports derived status filters:
  - `all`
  - `notstarted`
  - `inprogress`
  - `completed`
  - `favourites`
  - `hidden`
- supports sorting by:
  - last access
  - course name
  - short name
  - latest enrolment
- supports explicit sort order via `sortorder`
- supports display modes:
  - `card`
  - `list`
  - `summary`
- supports optional Back button rendering through `returnurl`
- supports runtime page title override through `title`

## Installation

1. Copy the plugin into `local/mycoursesfilter`
2. Visit `Site administration > Notifications`
3. Review the plugin settings
4. Add links to `/local/mycoursesfilter/index.php` where needed

## Plugin settings

The plugin provides these site-level settings:

- **Toolbar persistence**
  - `none`: do not persist toolbar selections
  - `core`: reuse compatible `my/courses` core preferences
- **Default category scope**
  - `recursive`: selected categories include descendants
  - `only`: only explicitly selected categories are used
- **Default page title**
  - when empty, the language string is used
- **Allow title override from URL**
  - when enabled, the `title` URL parameter may override the configured default

## URL parameters

Base URL:

```text
/local/mycoursesfilter/index.php
```

### Supported parameters

| Parameter | Type | Description |
|---|---|---|
| `coursename` | text | Case-insensitive partial match against course fullname and shortname |
| `tag` | text/integer | Course tag name or numeric tag id |
| `catid` | text | Category selector; supports ids, comma-separated lists, and `this`, `parent`, `children` |
| `customfield` | text | Combined custom field selector in the form `shortname` or `shortname:value` |
| `filter` | text | One of `all`, `notstarted`, `inprogress`, `completed`, `favourites`, `hidden` |
| `sort` | text | One of `lastaccess`, `coursename`, `shortname`, `lastenrolled` |
| `sortorder` | text | One of `asc`, `desc`; defaults depend on `sort` |
| `view` | text | One of `card`, `list`, `summary` |
| `returnurl` | local URL | Optional safe Back target; also accepts `this` |
| `title` | text | Optional page title override when enabled in plugin settings |
| `courseid` | integer | Optional explicit source course id for contextual category keywords |
| `only` | bool | Forces non-recursive category interpretation |
| `recursive` | bool | Forces recursive category interpretation |

### Compatibility notes

For migration convenience, the plugin still accepts these older request shapes
internally when the new parameter is missing:

- `q` as fallback for `coursename`
- `field` plus `value` as fallback for `customfield`

The legacy `status` alias has been removed.

### Category behaviour

`catid` accepts:

- a single id, for example `catid=2`
- multiple ids, for example `catid=2,3,4,5`
- contextual keywords:
  - `this`: the category of the source course
  - `parent`: the parent category of the source course category
  - `children`: the direct child categories below the source course category
- mixed forms, for example `catid=parent,12,children`

By default, category interpretation follows the plugin setting:

- `recursive` includes selected categories and all descendants
- `only` includes only the explicitly selected categories

Request switches override the default:

- `only=1` disables recursive expansion
- `recursive=1` forces recursive expansion

If both are present, `only` wins.

### Custom field behaviour

`customfield` supports two forms:

- `customfield=deliverymode`
  - matches any non-empty value in the field `deliverymode`
- `customfield=deliverymode:online`
  - matches when the field `deliverymode` contains `online`
    case-insensitively as a partial match

### Sort order behaviour

When `sortorder` is omitted, the plugin uses sensible defaults:

- `lastaccess` → `desc`
- `lastenrolled` → `desc`
- `coursename` → `asc`
- `shortname` → `asc`

### Return URL behaviour

`returnurl` is optional. When present and valid, the page renders a Back button.

Supported forms:

- explicit local path, for example `returnurl=%2Fmy%2Fcourses.php`
- contextual referrer mapping, `returnurl=this`

`returnurl=this` maps to the current page referrer only when the referrer
belongs to the same Moodle site.

### Title behaviour

The page title is resolved in this order:

1. `title` request parameter, when URL title overrides are enabled
2. configured plugin default title
3. language string fallback

## Example URLs

Filter by course name:

```text
/local/mycoursesfilter/index.php?coursename=biology
```

Filter by multiple categories:

```text
/local/mycoursesfilter/index.php?catid=2,3,4,5
```

Use contextual category selection from the current course:

```text
/local/mycoursesfilter/index.php?catid=parent&returnurl=this
```

Force non-recursive category matching:

```text
/local/mycoursesfilter/index.php?catid=12&only=1
```

Override the page title:

```text
/local/mycoursesfilter/index.php?filter=inprogress&title=Current%20learning
```

Filter by custom field and show a Back button:

```text
/local/mycoursesfilter/index.php?customfield=deliverymode:online&returnurl=%2Fmy%2Fcourses.php
```

## Access model

The page requires an authenticated user and checks the capability
`local/mycoursesfilter:view` in system context.

The plugin is **not** tied to a hard-coded learner role. It works with any role
model as long as the user is authenticated, has the capability, and is enrolled
in the returned courses.

## Status filtering logic

The status filter combines Moodle completion metadata and fallback access signals:

- `completed`: course completion exists or core completion flags it as completed
- `notstarted`: no start/completion metadata, or no recorded access when
  completion data is unavailable
- `inprogress`: started but not completed, or recorded access without completion
  when completion data is unavailable
- `all`: no status restriction

These are practical UI buckets derived from the best available metadata.

## Rendering approach

The plugin uses dedicated Mustache templates in
`local/mycoursesfilter/templates/` and exports a compact context in PHP for the
three supported display modes. The visual goal is to stay close to
`my/courses.php` while still allowing request-driven pre-filtering.
